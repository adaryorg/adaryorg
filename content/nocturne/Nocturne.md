# [[Nocturne]]
## what is Nocturne?
I'm still trying to figure out what is nocturne even tho it's something I came up with, and it sort of encompasses my work philosophy when it comes to work with one of the three (lets call the major) desktop platforms: Windows, MacOS and Linux. And when I say work, i mean the kind of work I do, which is mostly DevOps, development, system administration, and occasionally doodling and noodling with various pieces of code, software  and even hardware components. What I can define are probably the basic tenets of nocturne:

* Low distraction environment
* Dark and aesthetically pleasing desktop environment
* Sensible keyboard shortcuts for most if not all applications
* Using mouse as little as possible
* Utilizing as much screen real estate as possible for work

## why is Windows even on this list?
Before everyone starts crucifying me for putting Windows in the same bucket with Linux and MacOS, i want to state something obvious: I'm a working IT professional. One thing I can't afford is to be picky when it comes to working environments, and I for sure can't spend a lot of time customizing those environments to my taste.  In most cases the platform provided by my employer will be either Windows or MacOS, and I can only dream of working with Linux desktop (as long as i'm not messing around with my home setup which is mostly Linux based). And yes, Windows can be a perfectly viable work platform, it can provide tools that are fine for most tasks that need to be performed. <br>
Bottom line, we shouldn't be OS  snobs :)

## so what is my preferred platform to work on?
Last 15 years or so, i mostly did all my work on a Mac, did most my gaming on Windows, and played around usually in a home lab setting with Linux as my primary desktop. There were exceptions to this rule where I did have a Linux desktop at work, but that was for a very specific task, and the desktop was based on RHEL7 (back in 2016) so nothing glamorous or fun. I also had a few shorter periods where my work laptop was not a Mac, and I used it just fine for all the tasks I had at the time, which mostly involved connecting to remote systems and working there. In this scenario I don't really care if i use putty, linux/mac terminal or something third like vnc.

## what do i want to do with the nocturne idea?
The idea is to provide an easy way to adapt every desktop platform to Nocturne idea. Set a certain color theme, make sure that certain applications are installed, configured in a predictable way, and that the environment is completely familiar and usable from the get go. I am aware that something like this is relatively easy to achieve on Linux, moderately easy on MacOS, and probably very hard on Windows. That is the main reason that I'm starting everything with Mac (which  is my current laptop) and in parallel i'm fidgeting with the idea on Linux. A lot will be based on my collection of dotfiles and stow to manage them on both Mac and Linux, and beyond that i'm borrowing (or stealing?) ideas from the omakub project which is in my opinion absolutely awesome, but unfortunately restricted only to Ubuntu running Gnome.
